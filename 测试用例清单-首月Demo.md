# 测试用例清单：首月 Demo

## 1. 文档目标

本清单用于覆盖首月 Demo 的核心功能、接口、Agent 流程、数据质量和演示稳定性，确保对外演示时主链路稳定、可讲解、可录屏。

## 2. 测试范围

### 核心范围

- 首页与入口
- 对话式需求采集
- Agent 活动日志
- 方案生成
- 议价记录
- 一键下单
- 履约状态
- 会话记忆（当前 session）

### 扩展范围

- 购物广场
- 卖家控制台
- 长期记忆增强体验
- 结构化评价展示

## 3. 测试策略

1. 先保主链路稳定，再补展示页细节。
2. 优先测试高频 Demo 用例。
3. 优先验证“对外可演示性”，而不仅是功能存在。
4. 对 AI 输出设置结构化校验和兜底逻辑验证。

## 4. 功能测试清单

## 4.1 首页与入口

- 首页是否正常加载
- 价值主张是否清晰展示
- 首页 2D 视觉内容（平面示意、商品卡拼贴、2D 布局图）是否正常展示
- `Try Now` 按钮是否可点击
- 点击后是否进入对话页
- 是否未引入 3D 场景、3D 交互或实时 3D 渲染依赖导致异常
- 页面在主流桌面浏览器中显示是否正常

## 4.2 对话式需求采集

- 用户输入一句话需求后是否有正常响应
- 系统是否会追问预算、房间、风格等关键信息
- 用户输入不完整时是否有补充引导
- 用户中途修改条件时是否能更新上下文
- 多轮对话后是否形成结构化画像
- 异常输入时是否有兜底回复

## 4.3 Agent 活动日志

- 日志是否随着流程推进更新
- 是否展示检索、过滤、询价、议价、汇总等节点
- 文案是否符合用户理解
- 等待期间是否持续给予反馈
- 异常时是否展示友好提示

## 4.4 方案生成页

- 是否固定输出 3 套方案
- 每套方案是否包含商品清单、价格、推荐理由、空间效果图或布局建议
- 方案之间是否有明显差异
- 是否支持查看方案详情
- 若超预算，是否给出合理解释或替代方案

## 4.5 议价记录

- 是否可以打开议价记录
- 是否展示原价、折扣、最终价
- 折扣结果是否与方案价格一致
- 是否展示可解释的议价理由

## 4.6 一键下单

- 选择方案后是否可进入下单流程
- 下单按钮是否在正确条件下可点击
- 下单后是否生成订单号
- 是否进入订单成功页

## 4.7 履约状态

- 是否展示履约节点
- 履约节点顺序是否合理
- 文案是否可信、清晰
- 刷新页面后是否仍能查看订单状态

## 4.8 会话记忆（P0）

- 多轮对话中是否能记住前文预算、风格、房间等条件
- 用户中途修改条件后是否能正确作用于后续推荐
- 更新偏好后是否能作用于后续推荐

## 4.9 购物广场（如实现）

- 推荐卡片是否正常展示
- 是否有 Agent 战绩内容
- 是否体现基础个性化差异

## 4.10 卖家控制台（如实现）

- 页面是否完整展示底价、库存策略等字段
- 演示交互是否顺畅
- 文案是否能解释卖家 Agent 机制

## 4.11 长期记忆增强（如实现）

- 二次进入是否能识别用户偏好
- 是否能提及历史选择或偏好标签
- 是否符合产品预期，不影响主链路稳定性

## 5. 接口测试清单

## 5.1 会话接口

- 创建会话是否成功
- sessionId 是否正确返回
- 多轮消息是否关联到同一会话
- 非法 session 是否返回合理错误

## 5.2 Timeline / WebSocket 接口

- WebSocket 连接是否可正常建立
- Agent Timeline 是否按阶段实时推送
- 连接断开或超时时是否有合理提示与重试策略

## 5.3 推荐接口

- 完整需求是否可返回推荐结果
- 缺少关键字段时是否返回追问或提示
- 是否返回标准的 3 套方案结构
- 方案字段是否完整

## 5.4 议价接口

- 查询议价记录是否成功
- 折扣字段是否完整
- 异常商品是否有兜底结果

## 5.5 订单接口

- 创建订单是否成功
- 订单返回字段是否完整
- 查询订单状态是否成功
- 非法订单号是否提示正确错误

## 5.6 异常处理

- 模型超时
- 商品检索为空
- 会话失效
- 数据缺失
- 服务内部错误

以上异常都需要验证前端是否有可展示、可恢复的提示。

## 6. Agent 测试清单

## 6.1 需求理解

- 是否正确抽取预算
- 是否正确识别房间类型
- 是否正确识别风格偏好
- 是否正确识别特殊标签，如宠物、环保、耐抓
- 模糊输入是否能触发合理追问

## 6.2 方案生成

- 是否基于用户约束召回合理商品
- 是否能解释推荐理由
- 是否保证 3 套方案差异化
- 是否避免明显不搭配的组合

## 6.3 卖家 Agent

- 是否读取商品底价
- 是否考虑库存压力策略
- 最终价格是否不会低于底价
- 是否输出合理议价说明

## 6.4 多 Agent 协同

- 买家 Agent 与卖家 Agent 消息流是否正确
- 协同失败时是否能降级
- Timeline 是否与实际步骤一致

## 6.5 Prompt 稳定性

- 相同输入下结果是否相对稳定
- 英文场景文案是否自然
- 异常输入时是否避免明显失控回答

## 7. 数据测试清单

## 7.1 商品数据完整性

- 当前目录 `homary.json` 是否存在且可解析
- 商品总量是否符合预期（约 4500 条）
- 商品 ID 是否唯一
- 标题、价格、图片、类目是否完整
- 风格、材质、尺寸字段缺失率是否可接受
- 图片链接是否有效

## 7.2 数据规范性

- 价格格式是否统一
- 尺寸单位是否统一
- 风格标签是否标准化
- 类目字段是否标准化

## 7.3 推荐数据质量

- 候选商品是否与场景相关
- 商品组合是否符合常识
- 是否存在重复推荐
- 是否存在明显不适配商品

## 8. UI/UX 测试清单

- 页面视觉是否统一
- 按钮、间距、色彩是否一致
- Agent 文案风格是否统一
- 首页 2D 展示是否清晰、稳定、适合录屏
- loading 和等待态是否清晰
- 方案切换是否顺畅
- 弹层/侧栏是否好理解
- Chrome / Safari 下是否正常

## 9. 性能与稳定性测试清单

- 首页加载是否可接受
- 首页是否不存在 3D 资源或渲染逻辑导致的首屏阻塞
- 对话页首次进入是否流畅
- 方案生成总耗时是否符合 Demo 预期
- 连续多次演示是否稳定
- 页面刷新、返回、重进是否无严重问题
- 模型或接口异常时是否可降级

## 10. Demo 专项验收清单

### 10.1 主链路验收

- 首页 → Try Now → 对话 → 方案 → 议价 → 下单 → 履约 全链路可跑通
- 首页 2D 展示稳定，适合对外录屏
- 高频演示话术可稳定复现结果
- 页面切换流畅，适合录屏

### 10.2 讲故事能力验收

- 用户 30 秒内能理解产品做什么
- 用户 1 分钟内能感受到效率提升
- 用户 2 分钟内能理解 Agent 价值

### 10.3 录屏专项检查

- 页面无明显卡顿
- 演示账号、演示数据、演示脚本固定
- 外部不稳定依赖已降到最低
- 文案无明显错误、中英混杂或跳变

## 11. 推荐测试样例

### 样例 1：客厅搭配

- 输入：预算 3000 美元，原木风客厅，有两只猫，沙发需耐抓
- 预期：输出 3 套客厅方案，含耐抓沙发、价格解释、议价结果

### 样例 2：卧室搭配

- 输入：预算 2000 美元，现代简约卧室，需要环保材质，月底前送达
- 预期：输出符合风格和环保要求的卧室组合，并体现交付约束

### 样例 3：用户修改需求

- 初始输入：书房搭配，预算 1500 美元
- 追加入参：桌子换深色、增加收纳、不要金属感太强
- 预期：系统更新约束并重新生成更符合条件的方案

## 12. 测试通过标准

### P0 通过标准

- 主链路全部通过
- 高优先级接口无阻塞性缺陷
- 关键演示案例可连续稳定运行
- UI、文案、数据无明显硬伤

### 上线 Demo 前必须完成

- 主链路回归测试
- Demo 专项验收
- 录屏彩排至少 2 轮
