
## 一、架构总览

```
┌─────────────────────────────────────────────────────────┐
│                    Next.js 前端 (Vercel)                  │
│  ┌──────────┐  ┌──────────────┐  ┌───────────────────┐  │
│  │ 对话界面  │  │ Agent活动日志 │  │ 商品/方案展示区   │  │
│  └────┬─────┘  └──────┬───────┘  └────────┬──────────┘  │
│       │               │                    │             │
│       └───────────────┼────────────────────┘             │
│                       │ WebSocket + REST                 │
└───────────────────────┼─────────────────────────────────┘
                        │
┌───────────────────────┼─────────────────────────────────┐
│              FastAPI 后端 (Python)                        │
│  ┌────────────────────┴────────────────────────────┐    │
│  │            Agent Orchestrator (LangGraph)        │    │
│  │  ┌──────────────┐          ┌──────────────┐     │    │
│  │  │  Buyer Agent  │◄─博弈──►│ Seller Agent  │     │    │
│  │  │ (需求理解/    │          │ (定价/库存/   │     │    │
│  │  │  方案推荐)    │          │  折扣策略)    │     │    │
│  │  └──────┬───────┘          └──────┬───────┘     │    │
│  └─────────┼─────────────────────────┼─────────────┘    │
│            │                         │                   │
│  ┌─────────▼─────────────────────────▼─────────────┐    │
│  │              数据与记忆层                          │    │
│  │  PostgreSQL │ pgvector │ Redis (session/cache)    │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

**关键技术选型理由**：

| 组件 | 选型 | 理由 |
|------|------|------|
| Agent框架 | LangGraph | 支持多Agent状态机、可观测性好、流式输出 |
| LLM | GPT-4o / Claude Sonnet | 需求理解和方案生成，按需切换 |
| 向量数据库 | pgvector (PostgreSQL扩展) | 少一个独立服务，4500条数据量完全够用 |
| 实时通信 | WebSocket (FastAPI) | Agent活动日志需要实时推送 |
| 前端 | Next.js 14 App Router | SSR + 流式渲染，投资人演示加载速度快 |
| 部署 | Vercel (前端) + Railway/Fly.io (后端) | 快速部署，Demo阶段无需自建基础设施 |
