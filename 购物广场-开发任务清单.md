## 购物广场开发任务清单

## 1. 模块定位

- 所属项目：多 Agent 家居电商 Demo
- 在总项目中的优先级：**P1 展示型模块**
- 开发原则：不阻塞首页 / 对话页 / 方案页 / 下单页主链路
- 目标交付：一个可展示、可导流、可复用现有商品数据的购物广场页面

## 2. 交付目标

1. 完成 `/plaza` 页面
2. 展示商品发现、促销、Agent 战绩、结构化评价内容
3. 复用 `homary.json` 与现有后端能力
4. 支持基于 session 标签的轻量个性化
5. 完成到 `/chat` 的导流联动

## 3. 任务拆解

### 3.1 产品与设计

#### P0

- 明确购物广场信息架构与区块顺序
- 输出 Banner、商品卡、战绩卡、评价卡、唤醒卡样式
- 定义 CTA 文案与导流路径
- 定义空状态、加载态、错误态样式
- 统一“Agent 战绩展示区”的内容口径

#### P1

- 细化专题页/查看更多页原型
- 设计更强的个性化入口和筛选器

### 3.2 前端开发

#### 页面与组件（P0）

- 新增 `/plaza` 路由页面
- 开发 Banner 组件
- 开发商品卡组件
- 开发 Agent 战绩卡组件
- 开发结构化评价卡组件
- 开发智能唤醒卡组件
- 开发区块容器与响应式布局

#### 交互与状态（P0）

- 接入商品列表获取逻辑
- 支持按标签/类目分区展示
- 支持 session 偏好驱动的排序
- 处理 loading / empty / error 状态
- 实现卡片点击跳转 `/chat` 并附带预填参数

#### 增强项（P1）

- 增加筛选与排序交互
- 增加“查看更多同风格/同空间”交互
- 增加专题聚合模块

### 3.3 后端开发

#### 数据接口（P0）

- 提供购物广场商品聚合接口
- 提供今日上新 / 促销 / 推荐分区接口
- 提供 Agent 战绩流数据接口（可模板生成）
- 提供结构化评价展示接口或字段映射
- 提供广场导流参数协议

#### 个性化支持（P0）

- 从 Redis session 读取兴趣标签
- 提供轻量排序规则：类目偏好、风格偏好、预算偏好
- 无偏好时返回默认精选结果

#### 增强项（P1）

- 定时刷新推荐池
- 增加唤醒卡动态生成逻辑
- 接入长期记忆增强接口

### 3.4 数据与内容治理

#### P0

- 从 `homary.json` 提取适合广场展示的字段
- 补齐商品标签：风格、空间、材质、促销、卖点
- 准备默认专题：客厅、卧室、餐厅、猫家庭、原木风等
- 准备 Agent 战绩文案模板
- 准备结构化评价字段模板

### 3.5 联动与导流

#### P0

- 定义广场卡片到 `/chat` 的 query 参数协议
- 支持从商品卡进入“围绕这件商品继续聊”
- 支持从战绩卡进入“生成类似方案”
- 支持从专题卡进入“按该风格继续搭配”

### 3.6 QA 与验收

#### P0

- 验证 `/plaza` 页面可访问
- 验证各区块正常展示
- 验证商品数据读取正确
- 验证有/无偏好两种场景的展示差异
- 验证 3 类 CTA 均能正确导流到 `/chat`
- 验证 Demo 录屏时页面稳定

## 4. 推荐接口草案

- `GET /api/plaza/home`
- `GET /api/plaza/sections`
- `GET /api/plaza/achievements`
- `GET /api/plaza/wakeups`

返回建议统一沿用项目标准响应：

- `code`
- `message`
- `data`

## 5. 优先级清单

### 5.1 当前必做（模块首期）

1. `/plaza` 页面结构与视觉稿
2. 商品数据映射与展示卡片
3. Agent 战绩流展示
4. 结构化评价展示
5. 基于 session 标签的轻量排序
6. 导流到 `/chat` 的联动

### 5.2 后续增强

1. 定时上新机制
2. 专题页与更多筛选
3. 智能唤醒动态化
4. 与长期记忆增强联动

## 6. 依赖关系

### 强依赖

- `homary.json` 商品字段清洗与标签标准化
- 前端路由与 `/chat` 页面可用
- Redis session 中有基础偏好标签

### 弱依赖

- PostgreSQL / pgvector 可用于后续扩展，不是首期静态展示强依赖
- WebSocket 可用于后续实时内容刷新，不是首期必需

## 7. 验收口径

- 购物广场可作为独立展示页演示
- 页面内容与根目录 PRD/计划书口径一致
- 不引入全网搜索、真实社区、真实交易复杂闭环
- 能稳定承担“展示 + 种草 + 导流”职责
