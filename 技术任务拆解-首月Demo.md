# 技术任务拆解：首月 Demo

## 1. 文档目标

本清单用于将《多 Agent 家居电商 Demo》拆解为可执行的研发任务，便于排期、分工、跟踪和验收。

## 2. 交付目标

在首月内完成一个可稳定演示的 Web Demo，跑通以下主链路：

1. 用户输入自然语言需求
2. 买家 Agent 理解需求并触发商品检索
3. 卖家 Agent 根据底价和库存策略返回议价结果
4. 系统输出 3 套组合方案
5. 用户查看议价记录并一键下单
6. 页面展示订单与履约状态

## 3. 技术工作分层

### 3.1 产品与设计

#### P0

- 输出首页（2D 展示）、对话页、方案页、订单页高保真稿
- 输出 Agent 活动日志交互样式
- 输出议价记录展示样式
- 输出履约状态卡片样式
- 统一 Agent 人设、状态文案、交互文案
- 准备首页 2D 视觉素材与方案页空间效果图素材
- 明确本期不做 3D 场景、3D 交互与实时 3D 渲染

#### P1

- 设计购物广场页面
- 设计卖家控制台页面
- 设计结构化评价展示模块

### 3.2 前端开发

#### 基础搭建

- 初始化 Next.js 14 App Router 前端工程
- 配置路由、状态管理、REST / WebSocket 接口层
- 配置基础样式规范和组件体系

#### 页面开发（P0）

- 首页（2D 展示）
- Try Now 入口按钮与跳转逻辑
- 对话页
- Agent 活动日志组件
- 进度条/等待态组件
- 方案结果页（空间效果图 / 2D 布局建议）
- 议价记录弹层或侧边栏
- 订单确认页
- 订单成功页
- 履约状态页

#### 交互与体验（P0）

- 处理多轮对话交互
- 处理 loading / error / retry 状态
- 实现方案切换与详情展开
- 优化录屏场景下的动画和节奏

#### 展示页（P1）

- 购物广场页
- 卖家控制台页

### 3.3 后端开发

#### 基础服务（P0）

- 初始化 FastAPI 后端服务
- 设计 REST API 与 WebSocket 路由结构
- 配置日志、异常处理、中间件
- 配置 Redis session/cache，并管理演示用户状态

#### 业务服务（P0）

- 需求结构化服务
- 用户 Profile 更新服务
- 商品检索与过滤服务
- 商品组合方案生成服务
- 议价服务
- 订单创建服务
- 履约状态模拟服务

#### 推荐 API 草案

- `POST /api/chat/start`
- `POST /api/chat/message`
- `GET /api/agent/timeline/:sessionId`
- `POST /api/recommend/generate`
- `GET /api/recommend/:sessionId`
- `GET /api/negotiation/:schemeId`
- `POST /api/order/create`
- `GET /api/order/:orderId`

### 3.4 Agent 与 AI 编排

#### Agent Orchestrator（P0）

- 基于 LangGraph 搭建多 Agent 编排状态机
- 统一买家 Agent、卖家 Agent、检索服务之间的状态流转
- 对接 Timeline 实时事件输出

#### 买家 Agent（P0）

- 解析用户自然语言需求
- 提取预算、风格、空间、约束条件
- 组织候选商品召回
- 生成 3 套方案
- 输出方案解释文案

#### 卖家 Agent（P0）

- 读取商品底价与库存策略
- 根据规则给出折扣结果
- 输出议价说明
- 确保价格不低于设定底价

#### 编排能力（P0）

- 定义买家 Agent / 卖家 Agent 消息协议
- 定义状态流与 Timeline 文案映射
- 定义异常场景降级逻辑
- 为演示高频场景准备 Prompt 模板与默认结果

### 3.5 数据与检索

#### 数据治理（P0）

- 清洗当前目录 `homary.json` 中的 Homary 4500 商品数据
- 对价格、尺寸、材质、风格、类目字段做标准化
- 补齐基础标签
- 校验图片链接和关键字段完整性
- 将可检索字段写入 PostgreSQL / pgvector

#### 检索能力（P0）

- 关键词检索
- 类目过滤
- 风格过滤
- 预算过滤
- 特殊约束过滤（如宠物、环保、耐抓）

#### 组合规则（P0）

- 定义房间维度的商品组合模板
- 定义预算分配规则
- 定义方案差异化策略：性价比优先 / 风格统一 / 品质优先

### 3.6 订单与履约模拟

#### P0

- 定义订单数据结构
- 生成订单号与下单结果
- 定义履约状态模板
- 输出履约展示卡片

### 3.7 会话记忆与长期记忆增强

#### P0 会话记忆

- 基于 Redis 保存 session 上下文
- 支持用户在当前会话内修改条件
- 保存基础偏好标签

#### P1 长期记忆增强

- 预留用户长期记忆图谱/向量库接口
- 支持再次进入时的欢迎语和简单推荐

### 3.8 Demo 稳定性与演示模式

#### P0

- 为关键链路准备兜底数据
- 增加超时与降级机制
- 对主要案例做缓存或预生成
- 固定演示账号、演示脚本、演示用例
- 支持录屏友好的演示模式

## 4. 分优先级任务清单

## 5. 依赖关系

### 强依赖

- 页面原型 → 前端开发
- 商品数据清洗 → PostgreSQL / pgvector 导入 → 检索服务 / 推荐逻辑
- 需求结构定义 → Agent 编排 / API 设计
- 方案输出结构 → 前后端联调

### 弱依赖

- 购物广场可在主链路完成后补齐
- 卖家控制台可先做静态展示

## 6. 验收标准

### 主链路验收

- 首页 2D 展示与 Try Now 入口可正常进入
- 能完成一次完整需求输入
- Timeline 能展示 Agent 工作过程
- 能稳定生成 3 套方案
- 能查看议价记录
- 能完成一键下单并进入订单页

- 核心接口可用
- 关键页面无阻塞性错误
- 演示用例 3 次连续运行稳定
- 异常场景有兜底反馈
